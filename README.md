# conf_rout

Проект содержит два отдельных инструмента для генерации конфигураций роутеров на базе шаблонов:

- `KN-1112` — генератор конфигураций для Keenetic KN-1112
- `SNR-W4N` — генератор конфигураций для SNR-CPE-W4N

## Структура проекта

- `KN-1112/`
  - `make_keen.py` — скрипт генерации конфигураций на основе шаблона `KN-1112.txt`
  - `KN-1112.txt` — шаблон Keenetic-конфига с плейсхолдерами
  - `test_configs/` — папка для сгенерированных тестовых конфигураций

- `SNR-W4N/`
  - `make_snr.py` — скрипт генерации конфигураций на основе шаблона `SNR-W4N.dat`
  - `SNR-W4N.dat` — шаблон SNR-конфига с плейсхолдерами
  - `test_configs_snr/` — папка для сгенерированных тестовых конфигураций

## Как работает

Оба скрипта:

1. Запрашивают у пользователя данные абонента:
   - логин PPPoE
   - пароль PPPoE
   - имя Wi-Fi (`SSID`)
   - пароль Wi-Fi (`WPA-PSK`)
2. Читают шаблон из текущей папки
3. Подставляют значения в шаблон
4. Сохраняют итоговый файл в папку `test_configs` или `test_configs_snr`

## Формат шаблонов

В шаблонах используются маркеры Python `str.format`:

- `{LOGIN_PPPOE}` — логин PPPoE
- `{PASSWD_PPPOE}` — пароль PPPoE
- `{WIFI_NAME}` — SSID Wi-Fi
- `{WIFI_PASSWD}` — пароль Wi-Fi

## Запуск

Для `KN-1112`:

```powershell
cd d:\Code\conf_rout\KN-1112
python make_keen.py
```

Для `SNR-W4N`:

```powershell
cd d:\Code\conf_rout\SNR-W4N
python make_snr.py
```

## Результат

- `KN-1112/test_configs/test_config_{login_pppoe}.txt`
- `SNR-W4N/test_configs_snr/SNR_{login_pppoe}.dat`

> Обратите внимание: в `make_keen.py` присутствует общий блок кода для реальной отправки конфига на роутер через HTTP/RCI, но он закомментирован. В текущей версии скрипт выполняет только генерацию и сохранение тестового файла.

## Зависимости

Для одиночной генерации дополнительных зависимостей не требуется.

Для функционала сетевой отправки (если раскомментировать соответствующий код) потребуется:

- Python 3
- `requests`

Установка `requests`:

```powershell
python -m pip install requests
```

## Примечания

- Убедитесь, что шаблон находится в той же папке, где запускается скрипт.
- Сгенерированные файлы создаются с именем, включающим PPPoE-логин, чтобы не перезаписывать предыдущие результаты.
- Проверьте итоговые конфигурации перед заливкой на устройство.
